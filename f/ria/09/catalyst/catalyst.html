Flash Catalyst

Инструмент, существенно упрощающий взаимодействие дизайнера и программиста при работе над Flex-проектом.

Что может Flash Catalyst?

На данный момент инструмент имеет возможность импорта artwork из Photoshop (.psd) и Illustrator (.ai). Результатом импорта становится набор файлов на языке разметки MXML (.mxml), изображений (.png, .jpg) и файлов с векторной графикой на языке FXG (аналог SVG).

Данный набор файлов визуально компилируется в SWF-файл, который при просмотре представляет из себя почти то же, что было в исходных .psd/.ai. Инструмент позволяет добавить минимальную интерактивность, скроллбары, анимацию.

Проект хранится в одном файле. Программист импортирует данный файл во Flex Builder, где и ведутся работы по написанию кода.

Кто использует Flash Catalyst?

Может - дизайнер (при некотором обучении), может - программист.

Если провести вполне уместную аналогию с PSD -> HTML -> PHP, то Catalyst используется верстальщиком при переходе PSD -> HTML. Ясно, что как немногие дизайнеры занимаются версткой, так же немногие из них будут заниматься созданием "верстки" в Flash Catalyst.

В любом случае рабочий процесс сильно зависит от специфики каждого проекта. Существуют подходы, когда "верстать" вообще нет необходимости.

Заменяет ли Flash Catalyst другие инструменты?

Flash IDE

Нет. Flash IDE работает с таймлайном, графикой, изображениями. Во Flash IDE идет процесс "творения", в Catalyst - преобразования.

Flex Builder

Нет. Flex Builder - это среда для написания кода, сборки, отладки и профилирования приложения. Flash Catalyst тоже работает с MXML, однако работа идет не через текстовый редактор, а через визуальный.

Текущее положение дел

В следующей таблице описан рабочий цикл Flex-проекта с акцентом на взаимодействие дизайнера и программиста.

  Шаг Исполнитель Input Действия Output
1 Постановка задачи Руководитель Устное / письменное описание задачи
2 Дизайн Дизайнер Output предыдущего шага и этого шага (если он выполнялся ранее) Разработка дизайна, проектирование интерфейсов Файлы .ai, .psd, .fla и картинки
3 Обработка / экспорт Программист Output предыдущего шага и следующего шага (если есть) Преобразование .ai и .psd в .fla и картинки
4 Программирование Программист Output всех предыдущих шагов и этого шага (если он выполнялся ранее) Написание кода, реализующего логику приложения. "Оживление" дизайна. Файлы .fla, картинки, .css, .as, .mxml, ...

После Шага 4 снова переходим на Шаг 2.

Видно, что .ai и .psd файлы, преимущественно используемые на шаге дизайна, не могут быть использованы программистом. Их содержимое вручную экспортируется в .fla и картинки.

Если дизайнер будет использовать исключительно .fla и картинки, то задача упрощается. В этом случае при некоторых усилиях становится возможна совместная работа над одними и теми же файлами, т.н. Round Trip.

Недостатки этой схемы, устраняемые с помощью Flash Catalyst:

  • Необходимо постоянно тратить время на Шаг 3 - внесение изменений в дизайн занимает длительное время
  • Дизайнер не может легко добавить минимальную интерактивность в свой макет для того, чтобы оценить самому и продемонстрировать остальным постейший прототип приложения
  • Дизайнер не имеет доступа к анимации, создаваемой программно в .mxml

Предполагаемый рабочий цикл при использовании Flash Catalyst

Шаг 3 не требуется. В Output дизайнера появляются файлы .mxml и .fxg.

  Шаг Исполнитель Input Действия Output
1 Постановка задачи Руководитель Устное / письменное описание задачи
2 Дизайн Дизайнер Output предыдущего шага и этого шага (если он выполнялся ранее) Разработка дизайна, проектирование интерфейсов Файлы .ai, .psd, .mxml, .fxg, .fla, картинки
Шаг 3 не требуется
4 Программирование Программист Output всех предыдущих шагов и этого шага (если он выполнялся ранее) Написание кода, реализующего логику приложения. "Оживление" дизайна. Файлы .fla, картинки, .css, .as, .mxml, ...

.fxg файлы - имеют формат FXG 1.0. Это новый XML формат для работы с графикой для Flash Platform. Он описывает высокоуровневые графические и текстовые примитивы.

Резюме

Flash Catalyst сейчас далек от релиза и его применимость в рабочем процессе до конца ясна. Возможно, через 6-12 месяцев он займет свое место как средство для верстки .psd/.ai.

Пока же стадию верстки можно выполнить другими средствами или вовсе обойти.
